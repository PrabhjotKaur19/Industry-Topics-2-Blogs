Function Composition in JavaScript with Pipe

October 4, 2017

JavaScript is a very powerful language and it supports functional programing quite well. Function Composition toll make JavaScript even more powerful. I hope in this article, I will provide you an easy example of function composition with the use of pipe in JavaScript. So, you can understand function composition in JavaScript and you can write short, clean and functional code.

What is Function Composition?

Let’s start from basic by answering the question that what is function composition? Function composition is a mathematical concept to combine two or more functions it means result of one function is the input of the second function and so on. Function composition produce a new function as a result. It is like a pipeline in which data flows from one function to another function. Function composition works from right to left. For example ‘a()’ and ‘b( c )’ are two composable functions. When we do composition, this will like:

a(b(c))

this will work from right to left. That is why, in the above example the value that will be returned by function ‘b( c )’ will be passed to ‘a’. We can also compose more than two functions:

f5(f4(f3(f2(f1)))))

To further explain it let’s take a simple example in which we want to divide a number by 2 and then increment the result of division by 1. So, the code will look like this: 

     var num =4;
     var d= num/2;

     var result = d+1;
     console.log("Result is: ",result);
Imagine we have two functions for operations of division and increment. 
    var num=4;
   const division = (n) => {
   return n/2;

   }
   const increment = (a) => {
   return a+1;
   }
    var d=division(num);
    var result=increment(d);
    console.log("Result is: ",result);
In this way we need to create an extra variable ‘d’ to store the value return by division function and pass this value to increment function. There is nothing wrong with this approach because we have only two functions. But what if we have more than two functions? Then we need to declare many variables. We can solve this problem by the use of a concept called nesting functions. In which we can call one function from another function. This will look like as follows: 
    var num=4;
    const division = (n) => {
    return n/2;

    }
    const increment = (a) => {
    return a+1;
    }
    var result=increment(division(num));
    console.log("Result is: ",result);
This approach looks easy to read right now. But it will be confusing when we have multiple functions. To make this more readable JavaScript introduce the use of function composition with pipe.

What is pipe function in JavaScript?

In general, as the name suggested that pipe function create a pipeline of functions in which data flows. The meaning of data flows is that the output of a function is passed to next function and so on. At the end pipe function returns a function as a result. Pipeline of functions should look like as below:

function1 | function2 | function3 | ... | function n

The | symbol symbolize the pipe. The JavaScript function pipe is a very useful function which provides features like converts a list of functions into a function, supports array and multiple arguments.

Now, as we somewhat clear about the definition of the pipe function, we can start writing a pipe function. So that you can understand it more with a simple example that we use in this blog before at the start. 

    const pipe = (fun1,fun2)=>{
    return(arg)=>{
    const r = fun1(arg);
    return fun2(r);
    }
    }
    const result=pipe(division,increment);
    console.log("Result is: ",result);
Let’s breakdown the above code,

  • As I discussed above that division function accept a number as an input returns the output after dividing this number by 2.

  • The output of division is the input of increment that increments it by 1.

  • In the above function we passed these two functions to pipe in the order it should be executed. Pipe functions returns a new function result().

  • We call this result() from the console.log and passed a argument 4.

Refine pipe function

The problem with the above code that we still need to create a variable ‘r’ to store the return value of division(). But we can rewrite the current function like this: 

    const pipe = (fun1,fun2)=>(arg)=>fun2(fun1(arg));
So, in this way we do not need to use a separate variable and code is short, simple and easy to read.

Dealing with more than two functions

Our current function accepts two functions and return one function. But according to definition of pipe function, it accepts n number of functions and returns one function. To fit with the above definition function should be something like this: 

    const pipe = (fun1,fun2,fun3,...,funN)=>{
    return (arg) => funN(fun3(fun2(fun1(args))));
}
Continue with the above example, now we want to multiply the value returned by increment function with 100. To perform this operation, we have following function: 
    const multiply = (m) => {
    return m*100;
    }
As we know that pipe returns a function, we can also pass this returned function as an input to another pipe function, as shown below: 
    const finalValue = pipe(pipe(division,increment),multiply);
    cost result = finalValue(4);
In this way, I can nest as many pipe functions as many I want. But the problem is that it will become hard to read. For making our function easy to read we will need to do create a new pipe function. Our new pipe function is: 
     const pipe = (...funs) => funs.reduce(oldpipe)
Here, we uses … rest operator and a reduce function. The rest operator creates an array of functions and iterate over the array of operations by using reduce function and merge all operation into a single bundle. If we don’t pass an accumulator to reduce function than it will take first two elements of array to callback. These functions are then passed to oldpipe function. The function returned by oldpipe function becomes the accumulator. In this way reduce function iterates throughout the array of functions. With this new pipe function, we can pass n number of functions to it.

I have one more version of pipe function which is: 

    const pipe = (fun1,fun2)=>(...arg)=>fun2(fun1(...arg));
This pipe function accepts multiple arguments for first function. But it doesn’t work for second function because second function takes the value returned by first function and a function can return only one value.

Final Code:

const pipe = (fun1,fun2)=>(arg)=>fun2(fun1(arg));
const newpipe =(...funs)=>funs.reduce(pipe);
const division = (n) => {
return n/2;

}
const increment = (a) => {
return a+1;
}
const multiply = (m) => {
return m*100;
}
const finalvalue = newpipe(division,increment,multiply);
const r=finalvalue(4);
console.log("The result of ((4/2)+1)*100 is: ",r);

Explanation:

In this code, reduce function pass first two functions division and increment to oldpipe(). Which further call division and increment respectively and returned value 3 will be passed to multiply function. The result will be 300. So this is the simple example of pipe, I hope you enjoy reading this post and somewhat understand about pipe function.
