Introducing Ember.js1

Ember.js is an open source, free JavaScript client-side framework which designed to help you in developing web applications which are rich and complex in user interactions. It allows building client-side JavaScript applications by providing developers many features that are very important for managing complexity in these applications and as an integrated development toolkit. So, it provides a complete solution for web applications which contains data management and an application flow.

Ember.js uses the Model-View-Controller (MVC) architecture pattern. In this framework, the route is used as a model, handlebar template as view and controller manipulate the data in the model.

It allows developers to create scalable single-page web -applications. But it is also used for creating desktop and mobile applications (For Example Apple Music desktop application is an example of ember desktop applications). There are some popular websites that use Ember such as Discourse, Groupon, LinkedIn, vine, CodeSchool etc.

Ember.js was developed by Yehuda Katz and released in December 2011. The final version 2.10.0 of the Ember.js was released on November 28, 2016. It was first named as SproutCore MVC framework.

The major concept that team used in the creation of the Ember was the belief that web has the ability to bookmark and share URLs is what makes it so unique. The team felt that most of other JavaScript frameworks take the concept of URLs as a second thought such as they use the concept of a router as an optional add-on. They want to create a framework that is somewhat similar to the concept of desktop MVC framework instead of using the server-side MVC framework. That is why, they created a framework Ember.js that is a combination of the productivity of a native UI framework and share URLs. They designed the Ember.js with some following key ideas2:

Focus on ambitious web applications

Ember was designed to solve the problem of the client-side applications. This JavaScript framework first provides the solution to V in MVC(Model-View-Controller) and then try to grow further from there.

More productive out of the box

Ember.js is like a box with a set of tools. These tools work together to give you a complete development kit. The motive of these tools is to allow developers to work more efficiently.

Stability without stagnation

This concept ensures that backend compatibility is important and can be maintained while using the framework.

Future web standards foresight

Many of the standards of the JavaScript and the web has been supported by ember earlier than other frameworks such as web components, ES6 syntax etc. Ember also follows the principles of Convention over Configuration (CoC), and Don’t Repeat Yourself (DRY).

Features of Ember.js34

Following are some features of ember.js:

  • Routes are the main features of ember and the central part of any ember application. These routes are used to manage the URLs of applications.

  • Ember provides an Ember Inspector Tool to debug your ember applications. It is a browser extension that allows you to inspect ember applications live.

  • Ember uses handlebars syntax to write your applicationâ€™s templates which allows you to write less code and automatically update the model when the content of application get changed.

  • Ember also provides you a development toolkit Ember CLI to create, develop and build ember applications.

  • Ember Data Layer provides an efficient way to communicate with external APIs and also manage the application state.

  • Ember.js use HTML and CSS as the main development model and allows to create reusable and maintainable JavaScript applications.

Installing Ember.js5

Getting started with ember is quite easy, you just need to install ember command line interface. Ember CLI is used to create and manage ember applications through the command line. Ember CLI provides:

  • Different kinds of application asset management such as concatenation, minification, and versioning.

  • Generators to produce components, routes etc.

  • A typical project layout which makes easy to approach existing ember applications.

  • Supports ES2015/ES6 and modules of JavaSript. Dependencies You need to have following dependencies to install ember in your system:

  • Git: Git is open source system to control the versions of files. Ember uses git for managing its dependencies. For more information on git visit the official website of git. If you do not have git installed on your system, you can install from these links for Linux, for Mac and for Windows.

  • Node.js and NPM: Node.js is an open source, which is used for developing server-side and networking JavaScript applications. NPM is used to manage dependencies. Ember needs the latest version of it. If you do not have node.js installed visit these links Mac, Linux or Windows. NPM is packaged with Node.js.

  • Other Dependencies: For managing the components such as HTML, CSS, JavaScript, image files etc. you need Bower. You can install it using NPM. Other optional dependencies are Watchman and PhantomJS. Watchman is used to managing files and directories and do some actions when they change. PhantomJS is used to run the browser-based unit test. Now as you have all required dependencies installed on your system, you can install ember with npm by running this command given below:

npm install -g ember-cli

Or you can run the following command to install specific version:

npm install -g ember-cli@2.16

To verify that you successfully installed ember, run (for windows):

Ember -version

If the result of this command is version number than ember is installed on your system and you are ready to create applications with Ember.

Core Concepts of Ember.js6

Before you start developing ember applications you need to know about some basic concepts of Ember.js which are given below.

Source: https://www.tutorialspoint.com/emberjs/images/emberjs-core-concepts.jpg

  • Router and Route Handlers

    Whenever URL is changed router maps the URL to route handlers. Imagine user load the app by entering the URL in the address bar and change the URL by editing the URL in the address bar or by clicking a button in the app or any other way. The first thing that happens is that ember router maps the changed URL to route handler. Which further renders a template and loads a model for a template. If the user has no required permissions to access the specific part of the app router will redirect the user to the new route.

  • Templates

    The user interface is what known as templates in ember. These templates build the front-end application, which is similar to HTML and uses the syntax of Handlebars. It also allows to use and update regular expressions.

  • Model

    In ember applications, we use model is used to store persistent state. These models manipulate the data and templates to be displayed in HTML. Ember manipulates the data stored in the database by extending the functionality of the Ember Data.

  • Components

    As we know that templates control how the user interface looks like and components control how the user interface behaves. There are two parts included in components:

    • A template file which is written in Handlebars,

    • A source file which is written in JavaScript that defines the behavior of UI.

It allows the creation of reusable controls.

Creating and Running Application7

Now as you know the basic concepts of ember and how to install ember in your system. Letâ€™s start creating simple and easy ember application. First, create a folder (for example first-ember-app) where you want to store your application. Then navigate to that folder on the command line using cd command:

$ cd ~/first-ember-app

Now create a new application by running the following command:

$ ember new first-app

Now as you created an ember application, to run this application you need to navigate to this projectâ€™s directory as follows:

$ cd first-app

Now run,

$ ember server

Your application is running now. Open the browser and go to http://localhost:4200/. It will show the ember welcome page which means your application is running now. You can edit this application. For this open app/templates/application.hbs and make changes for example:

<h1>Welcome from my side!!</h1> 
{{oulet}}

As you can see ember automatically update the changes when you change this file and show updated page.

Define a route: Letâ€™s create an application that just displays the grocery list. For this we need to create a route, so run in your project directory:

$ ember generate route grocery

Do the following changes to app/templates/grocery.hbs

<h2>Grocery Items</h2>

You also need to define a model in app/routes/grocery.js file

import Route from '@ember/routing/route';
export default Route.extend({
model(){
return['Milk', 'Apple', 'Breads'];
}
});

Now tell the ember how to render that model in HTML. Add following into grocery.hbs file:

<h2>Grocery Items</h2>
<ul>
{{#each model as |grocrey|}}
<li>{{grocrey}}</li>
{{/each}}
</ul>
{{outlet}}

Now visit the http://localhost:4200/grocery to see changes. I hope you enjoy reading this blog and somewhat understand about ember. If you want to learn more about please see official guide of ember.
