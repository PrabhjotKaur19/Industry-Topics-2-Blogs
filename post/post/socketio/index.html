Welcome to my new post in which I am going to explain about Socket.IO JavaScript library for real-time web applications. In this article, I will introduce you to the basic concepts of Socket.IO. I hope you will enjoy reading this article and understand basics of this topic.

Introduction to Socket.IO1,2

Node.js is not the solution to every problem, however, using the Socket.IO library we can easily make it work with Websockets for real-time applications. Socket.IO is a JavaScript library. It enables real-time bidirectional event-based communication between web clients and web servers. It has two parts which are the client-side library and the server-side library. In this client-side library runs in the browser, on the other hand, the server-side library for Node.js. These both parts have identical API. The major feature of Socket.IO is that it works on every platform such as on different browsers or devices with equal reliability and speed. It is mostly used in analytics, document collaboration, streaming and instant messaging. Some of the users are Microsoft Office, Yammer, and Zendesk. Socket.IO uses WebSockets. If WebSockets are not available it will switch to something else that browser supports.

Socket.IO is a simple method to push messages from the client to the server and vice-versa. It also includes some important features such as broadcasts and multicast.

Real-time applications3

So, as we know that Socket.IO is used for real-time, so what are real-time applications exactly? A real-time application (RTA) that perform functions within a current period when a user is working. There are many real-time applications such as

  • Videoconference applications

  • VoIP(Voice Over Internet Protocol)

  • Online Gaming

  • Instant Messengers (chat apps like WhatsApp, Facebook etc.)

  • Collaboration Applications (for example: google docs which allow users to work on the same document and apply changes made by one user to another users document.)

Why Use Socket.IO?4

At this point as we know that what is Socket.IO, now the question is that why we should use this over other libraries for real-time applications? The answer is that when we create these applications with web applications stacks like a LAMP (PHP) has been very difficult and slow process as compared to Socket.IO. It includes different processes such as polling the server for changes, tracking the timestamps etc.

Most of the real-time applications are architected using the sockets which allow client and server to communicate with each other. In this way whenever an event occurs, the server will detect it and send messages to clients. Socket.IO is quite popular and used by many organization (Microsoft Office, Yammer, Trello, and Zendesk) to build applications. Socket.IO also has a huge community so getting help is easy.

It handles browser inconsistencies and varying support levels for you. It includes additional features beyond bare bones web sockets such as room support for basic publish/subscribe infrastructure and things like automatic reconnect. sockets are really fast and are capable of sending only those data packets which are needed.

For Example: Let’s imagine that you are working on a chat application. You want to implement the functionality that when user A is typing something then user B should know that A is typing, (similar Facebook messenger and WhatsApp). If you will use HTTP calls here then “B” will not be able to know about the actual status of another person because of the delay. So, the solution is that you can use sockets rather than HTTP calls. In this way when user A is typing anything then his device will send only one data packet which will just notify the server that he is typing and the server will deliver this to user B, this process is really fast (almost real-time) and will reduce the data transfer also.

Getting Started with Socket.IO56

Prerequisites:

Before you start developing with Socket.IO you need to install Node.js and npm on your machine. If you do not have these installed then install Node and NPM.

After you install node and npm, open the terminal and create a new directory by running following command:

$ mkdir testSocket

Now change the current directory to new directory:

$ cd testSocket

Now using the following command initialize the npm:

$ npm init -y

Here we use -y to set the default values in package.json. In this example, we are going to use Express. You can use any of the node framework or node HTTP server. But I think that ExpressJS is easy to use. So you need to install both ExpressJS and Socket.IO by running following command:

npm install –save express socket.io

Now we need to install nodemon so that you do not need to restart the server whenever you made any changes.

npm install -g nodemon

At this point, you are ready to develop your first app with socket.io. Create a file named index.js and copy the following code:

         var app = require('express')();

         var http = require('http').Server(app);


         app.get('/', function(req, res) {
         res.sendfile('index.html');
         });
         http.listen(3000, function() {
         console.log('listening on *:3000');
         });

You need a HTML file named index.html

<!DOCTYPE html>
<html>
   <head>
      <title>Hello world</title>
   </head>
   <body>Hello world</body>
</html>

To test app at this point

nodemon index.js

It will shows listening on *:3000. Now it is working at this point. You need to change the index.js file:

var app = require('express')();
var http = require('http').Server(app);
var io = require('socket.io')(http);

app.get('/', function(req, res) {
   res.sendfile('index.html');
});

//Whenever someone connects this gets executed
io.on('connection', function(socket) {
   console.log('A user connected');

   //Whenever someone disconnects this piece of code executed
   socket.on('disconnect', function () {
      console.log('A user disconnected');
   });
});

http.listen(3000, function() {
   console.log('listening on *:3000');
});

In this, we require a socket.io and set up our server to log messages when a user visits this page or left this page. After this, we need to add some code to index.html file to add the client-side script.

<!DOCTYPE html>
<html>
   <head>
      <title>Hello world</title>
   </head>
   <script src = "/socket.io/socket.io.js"></script>
   
   <script>
      var socket = io();
   </script>
   <body>Hello world</body>
</html>

Now if you visit localhost:3000 it will show “Hello World”. When you check logs it will show “A user connected”. At this point, you have your Socket.IO working. As you see it is very easy to setup Socket.IO. This is a basic of Socket.IO if you want to learn more about it you can visit the official documentation. I hope you enjoyed reading this article.
